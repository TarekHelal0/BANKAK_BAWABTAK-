 -Overview

A database system designed to help bank managers handle employee operations, requests, and transaction monitoring through a form-based interface.

 -Features
Hire and fire bank tellers
Manage employee records
Handle vacation requests (submit / approve / reject)
View and filter transactions
Form-based UI with strong input validation
 -Technical Highlights
Relational database with SQL (CRUD operations)
Constraints (Primary & Foreign Keys)
Strong data validation (forms + database level)
Structured and modular design
 -Interface
Simple forms for managing:
Employees
Requests
Transactions
